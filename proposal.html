Section 01

프로젝트 개요

연세대학교 마일리지 수강신청 시스템의 역대 신청 데이터(3~4년치)를 스크래핑하고, 각 과목별 합격 확률을 통계적으로 계산한 뒤, GPT-4o-mini (OpenRouter)가 총 마일리지 예산 안에서 우선순위를 반영한 최적 배분 전략을 Structured Output으로 생성한다.

핵심 아이디어

수강신청 마일리지 역대 기록은 로그인 없이 공개되어 있다. 이 데이터로 "과거에 X점 넣은 사람의 프로필이면 붙었는가"를 시뮬레이션할 수 있다. 단순 통계 수치를 LLM에게 넘겨 자연어 전략과 리스크 분석까지 함께 받는 것이 이 프로젝트의 차별점이다.

4-2  JS 통계 계산 로직

Python의 simulate_rank()를 JS로 포팅한다. 각 과목 × 역대 학기 × 마일리지 후보값에 대해 시뮬레이션을 반복해 합격률을 계산한다.

입력
내 프로필 + 마일리지 X, 역대 학기별 신청자 목록 (수강여부 Y인 사람 = 합격)
핵심 처리
내 레코드를 신청자 풀에 삽입 → 우선순위 기준 a~h로 정렬 → 내 순위 도출 → 정원 이내 여부 판별
합격률
합격한 학기 수 / 전체 학기 수 (마일리지 X를 1점씩 올려가며 반복)
출력
과목별 [ {mileage: X, probability: 0.xx} ] 배열 → LLM 프롬프트에 삽입

우선순위 정렬 기준 (a → h)

  • a 마일리지 높을수록 우선
  • b 특수교육대상자 우선
  • c 전공자 우선 (개설학과 설정)
  • d 총 신청과목수 많을수록 우선 (최대 6개 반영)
  • e 졸업/수료 신청자 우선
  • f 초수강자 우선
  • g 총이수학점 / 졸업이수학점 비율 높을수록 우선 (max 1.00)
  • h 직전학기 이수학점 / 수강가능학점 비율 높을수록 우선 (max 1.00)
  • i 동점 시 난수 부여

4-3  최적화 알고리즘

JS에서 1차 최적 배분을 먼저 계산하고, 그 결과를 LLM에 넘긴다.

목적함수
가중 합격확률 합산 최대화
Σ (weight[i] × P(course_i, mileage_i))
제약 조건 1
총 투입 마일리지 ≤ 총 보유 마일리지 (예: 72점)
제약 조건 2
과목별 투입 ≤ 과목별 최대 마일리지 (JSON에서 읽음, 예: 12점 / 36점)
우선순위 가중치
1순위: 6pt / 2순위: 5pt / … / 6순위: 1pt (조정 가능)
탐색 방법
Greedy + 브루트포스 조합 (과목 수 ≤ 6이라 현실적)

4-4  OpenRouter LLM 호출 (gpt-4o-mini)

JS 최적화 결과(배분안 + 확률 배열)를 프롬프트에 담아 LLM에 전달한다. JSON Schema Structured Output을 강제해 정형화된 응답을 받는다.

응답 JSON 스키마

JSON Schema · response_format
{
  "strategy_summary": "string",          // 전체 전략 한 줄 요약
  "total_mileage_used": "number",        // 실제 사용 마일리지 합계
  "courses": [
    {
      "priority":               "number",
      "course_code":            "string",
      "recommended_mileage":    "number",
      "acceptance_probability": 0.0~1.0,
      "risk_level":             "low | medium | high",
      "min_safe_mileage":       "number",  // 합격 기대 최소 마일리지
      "reasoning":             "string"   // 판단 근거
    }
  ],
  "warnings": ["string"],               // 리스크 경고 메시지 배열
  "alternative_distribution": {
    "description": "string",             // 보수적 배분 설명
    "courses": ["…same structure…"]
  }
}

LLM 역할 요약

  • → JS가 계산한 최적 배분이 합리적인지 검토 및 보완
  • → 리스크 분석 (경쟁이 급등하는 과목, 마일리지 쏠림 위험 등)
  • → 보수적 대안 배분 제시 (위험 분산)
  • → 자연어 전략 설명 (사용자 친화적 출력)
Section 05

Python 스크래퍼 수정 사항

기존 스크래퍼는 CSV 2개를 출력한다. HTML이 직접 파싱할 수 있도록 단일 JSON 출력으로 변경한다.

출력 JSON 구조 (과목 1개 당 1파일)

mileage_HUM2038-01.json
{
  "course_info": {
    "course_name":    "글쓰기",
    "course_code":    "HUM2038",
    "section":        "01",
    "max_mileage":    36,           // 이 과목에 투입 가능한 최대 마일리지
    "total_capacity": 30,
    "major_quota":    10            // 전공자 전용 쿼터 (없으면 0)
  },
  "semesters": [
    {
      "semester_name": "2024년 1학기",
      "summary": {
        "total_capacity": 30,
        "major_quota":    10
      },
      "applicants": [
        {
          "mlgVal":                   20,   // 투입 마일리지
          "dsstdYn":                  "N",  // 특수교육대상자
          "mjsbjYn":                  "Y",  // 전공자
          "aplySubjcCnt":             5,    // 총 신청과목수
          "grdtnAplyYn":              "N",  // 졸업신청
          "fratlcYn":                 "Y",  // 초수강
          "ttCmpsjCdtRto":            0.72, // 총 이수비율
          "jstbfSmtCmpsjCdtRto":      0.95, // 직전학기 이수비율
          "lsnYn":                    "Y"   // 수강여부 (Y=합격)
        }
      ]
    }
  ]
}
